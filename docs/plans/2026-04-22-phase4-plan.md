# Phase 4 Plan — Execution Mode Replication + Preuse Arms + Phase 5 Holdout

**Issue**: #329 Track E27 (exec-mode).
**Predecessor**: Phase 3 (N=400) — analyst report commit `472cc9f`, Codex cross-check `9c36973`, H8 F10 regrade `f5fdd3d`.
**Rule binding**: ADR `docs/adr/2026-04-22-rule-4-mode-selection.md` — Narrow Lock. Promotion to Full Policy Lock depends on this plan's successful execution.
**Status**: DRAFT — awaiting orchestrator + user approval.

---

## 1. Goal

Convert Narrow Lock → Full Policy Lock by:

1. **Replicating Phase 3** under widened seed distribution to confirm the S/D/Pfresh/Pacc rankings are stable, not sample artifacts.
2. **Evaluating Preuse arms** (Preuse-clear + Preuse-compact at 4 thresholds) to decide whether Rule 4-A Step 4 can activate.
3. **Validating generalization** via Phase 5 holdout (5 new fixtures) reaching ≥70% grader accuracy.
4. **Resolving tracker questions** Q1, Q2, Q7 whose answers are prerequisites for Phase 4 reproducibility.

Outcome: a Phase 4/5 dataset that either (a) promotes Rule 4-0 to Full Lock, (b) revises Rule 4-A (e.g., Layer 1 default flips), or (c) supersedes the ADR with new data.

## 4. Pre-registration

- **New tag** (required before any Phase 4 trial fires): `exec-mode-v4-replication-preregistered-YYYYMMDD`.
- **Scope of tag**: fixture set (Fa + F2–F10), seed list, mode set (D/S/Pfresh/Pacc + Preuse-clear + Preuse-compact × T1–T4), grader harness version.
- **Why separate from Phase 3 tag**: wider seed distribution and new Preuse arms are a different pre-registration. Reusing the Phase 3 tag would conflate sample expansions with mode additions.
- **Phase 5 tag**: additionally `exec-mode-v5-holdout-preregistered-YYYYMMDD`. Declares the 5 holdout fixtures and locks them against grader tuning.
- **Registration authority**: orchestrator (aigentry-orchestrator-claude) + user approval, committed to `docs/superpowers/specs/` before fixtures run.

## 7. Success criteria

Full Policy Lock is granted iff **all** of the following hold:

1. Phase 4b rankings (quality-weighted) preserve Phase 3 ordering within 1 rank, with effect-size stability (e.g., S vs D quality gap change <0.05 absolute).
2. Phase 4c Preuse-compact best-threshold beats Phase 3 baseline Pacc on the same fixtures by a pre-declared margin.
3. Phase 5 holdout grader accuracy ≥70%.
4. Pre-registration tags committed before fixture execution, with no post-hoc grader changes.
5. Orchestrator + user sign-off recorded as ADR status transition (Proposed → Accepted → Revised).

Any failure condition triggers ADR status **Superseded** and a new ADR.

## 8. Risks & mitigations

| ID | Risk | Mitigation |
|---|---|---|
| R1 | Substitute-compact summarizer drift (deterministic across replications? cross-fixture stable?) | Pin summarizer spec under Phase 4 pre-registration tag (ADR §4.6 `substitute-compact-v1`); verify byte-equality across two implementations on the 10-input regression set with SHA-256 digests before kickoff (V3 metric, hard reject on partial pass) |
| R2 | Q7 says no CLI supports auto-compact disable | Restrict Phase 4 to Claude (Rule 4-0 scope); annotate Full Policy Lock as Claude-only |
| R3 | Preuse-compact best threshold changes by fixture | Report per-fixture winners; if no single threshold generalizes, reject Preuse-compact from Rule 4-A Step 4 |
| R4 | Holdout accuracy <70% — grader brittleness | Grader audit (Rule 4-A Step 4.5 escalation path) before rerunning; do **not** retune graders on the holdout itself |
| R5 | Budget overrun (1,600 trials) | Serialize Phase 4b → 4c → 5; publish checkpoint after each; allow early-stop on Phase 4b ranking failure |
