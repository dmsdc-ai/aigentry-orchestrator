# 터미널 어댑터 소유권 분석 (6관점 + 3적대반박 + 종합)

**결론**: yes-with-nuance (confidence 0.93). 6관점 만장일치 yes, 적대반박 전부 consensus 생존.

## 한 줄 답
네, 맞습니다 — 단, 정확히는 "터미널 SURFACE를 구동(spawn/close/focus)하는 어댑터"가 오케스트레이터(bin/lib/workspace-host.sh 어댑터 seam) 소유입니다. telepty는 세션/PTY 존재 + read-only liveness probe + surface_orphaned 신호만 유지하며, 이 분할은 2026-05-30 ADR로 이미 결정·구현·테스트 완료된 상태(open question 아님)입니다.

## 분할
drive vs own-existence 선을 따라 분할됨 (ADR 2026-05-30 §1, 코드로 검증). SURFACE-DRIVING(보이는 터미널 창을 spawn/close/focus = UI 컨테이너 생명주기) = 오케스트레이터. 단일 seam workspace-host.sh가 cmux/warp/headless 5-method 계약(wh_lookup/close/alive/list_ids/focus + 추가 prune/set_status/clear_status)으로 소유 (검증: warp 어댑터 _wh_warp_close/_wh_warp_focus/_wh_warp_alive 모두 :334-392에 실재). SESSION-EXISTENCE/TRANSPORT(PTY spawn/resize/kill, 세션 레코드, inject=cmuxSendText 바이트 구동, read-only isSurfaceAlive probe, surface_orphaned 신호 emit, zombie-SESSION GC) = telepty(신경계, CONSTITUTION.md:57 '절대 안 함 = UI 렌더링'). 핵심 구분: 바이트를 surface에 쓰는 것(drive)=telepty / surface가 존재를 멈추도록 결정(own-existence)=오케스트레이터. creator=destroyer: open-session.sh가 surface를 생성하므로 그 소멸도 소유. 데이터 위치(telepty가 cmuxWorkspaceId 저장)는 PROBE(읽기) 소유만 정당화, ACTUATION(파괴) 소유는 아님 — wh_lookup이 공개 'telepty list --json' 계약으로 같은 필드를 깨끗이 읽음.

## 유일 nuance (§9)
§9 독립(standalone telepty) — 이것이 답을 진짜로 한정하는 유일한 tension. CONSTITUTION.md:132 'telepty는 orchestrator 없이도 동작한다'가 헌법적 구속이고, 오케스트레이터 없이 설치된 telepty가 surface를 절대 close 못하면 ghost 탭이 누적됨. ADR가 두 leak을 분리해 해결(검증 완료): (1) zombie SESSION leak(실제 자원/정확성 harm)=telepty의 session-GC가 무조건 reclaim(daemon.js:4072-4122, surface_orphaned emit 포함) → standalone에서도 §9 충족; (2) ghost SURFACE 탭(cosmetic)=managed 경로는 오케스트레이터 reconciler, standalone은 opt-in AIGENTRY_TELEPTY_SELF_CLOSE_SURFACE=1(default OFF)로 처리. §9는 telepty가 standalone으로 '동작'할 것을 요구할 뿐 'surface-close를 default 소유'할 것을 요구하지 않음. 게다가 telepty-측 close는 cmux 전용(6개 backend 중 1개)인 반면 어댑터 seam은 cmux+warp+headless 균일 커버 → §9 커버리지가 오히려 strictly 우월. (모네타이제이션 wedge='durable cross-machine SESSIONS'=telepty의 세션/PTY 지속성으로, surface-driving과 무관 → 제품 충돌 없음.)

## telepty leak
관리경로(managed) 누수는 없음 — closeSurface(terminal-backend.js:497-513)는 AIGENTRY_TELEPTY_SELF_CLOSE_SURFACE!=='1' → return true 게이트(default no-op)이고 호출처 daemon.js:1823 + daemon.js:3356은 명시적으로 §9 standalone fallback(검증된 ADR-sanctioned). focusSurface는 telepty에서 완전 제거됨(test/lifecycle-surface-acceptance.test.js:290-292가 typeof===undefined 단언). 단 ONE gray-zone 실재 surface-touch 발견: src/transport/websocket.js:88 — `kitty @ --to unix:${sock} set-tab-title --match id:${wid} '⚡ telepty :: ${sessionId}'` execSync. 이는 reconnect 시 탭 LABEL을 쓰는 것(open/close/focus 생명주기가 아닌 라벨링)이라 ownership 위반은 아니나, telepty 안에 남은 유일한 live surface-write이므로 #608이 '라벨링≠생명주기'로 명시 carve-out 할지 어댑터 seam 경유로 라우팅할지 결정해야 함. tui.js:162/194의 kitty send-text는 byte-driving(telepty 정당 역할). ⚠️ 중요 사실정정: ADR §9가 'closeSurface/focusSurface는 uncommitted working-tree, land 전 제거'라고 가정했으나 실제 telepty HEAD f4641b7는 working tree clean + rework가 이미 LANDED(closeSurface=게이트된 no-op로 commit, focusSurface 제거 완료). #608이 의존하는 ADR §9 sequencing 전제는 outdated.

## #608 함의
- 1. 계약의 component HOME = 오케스트레이터 workspace-host.sh seam으로 anchor. SPAWN을 telepty로 올리지 말고(절대) workspace-host.sh로 내려서 close/focus/alive와 통합 — 단 'surface-DRIVING'만. probe/transport/send-key를 telepty 밖으로 끌어내면 §3(신경계) 위반·ADR가 제거한 conflation 재도입.
- 2. 현재 코드의 split-brain을 #608이 해소: SPAWN은 open-session.sh:216-272 인라인 case(cmux/aterm/tmux/wezterm/iterm, **warp 부재**, workspace-host.sh를 source 안 함) vs LIFECYCLE는 workspace-host.sh(cmux/warp/headless, aterm/tmux/wezterm/iterm 부재). 두 절반이 backend 매트릭스가 다르고 코드 공유 안 함 → 이것이 relocation 이유가 아니라 unification 이유. 단 backend별 spawn 메커니즘이 본질적으로 다름(cmux new-workspace vs warp:// deeplink[Warp는 spawn CLI 없음] vs telepty spawn fallback) → 하나의 uniform spawn verb를 가정 말고 seam 뒤에 adapter-specific spawn 유지.
- 3. telepty-측 read-only touchpoint를 계약에서 명시 보존: isSurfaceAlive probe(terminal-backend.js:128) + surface_orphaned bus event(daemon.js:4105) + decideSurfaceGc + zombie-session GC. #608은 이를 telepty의 유일 surface 접점으로 못박고, 어댑터는 surface_orphaned를 consume해 wh_close.
- 4. §9 standalone floor 보존: AIGENTRY_TELEPTY_SELF_CLOSE_SURFACE=1(default OFF) opt-in을 managed default로 승격 금지(double-close 회피). standalone spawn은 telepty의 자체 primitive(telepty allow current-terminal wrap + kitty fan-out)가 §9 floor — #608이 이를 제거/relocation하면 monetization wedge 파손.
- 5. websocket.js:88 kitty set-tab-title cosmetic write를 #608이 명시 분류: '라벨링은 lifecycle 아님'으로 carve-out 하거나 어댑터 seam의 새 wh_label 메서드로 라우팅. 미결 시 boundary 회색지대 잔존.
- 6. ADR §9 sequencing 전제 갱신 필요: rework가 이미 landed(telepty HEAD clean). #608은 'staging uncommitted diff'가 아니라 '이미 land된 분할 위에 unified spawn 계약을 얹는' 작업으로 재프레이밍. 회귀 게이트는 #486 INV-17(cmux/warp 둘 다 app-quit→INDETERMINATE→alive로 mass-kill 방지: _wh_warp_alive:360-363, isSurfaceAlive 'unknown'→skip 확인 완료) 비-회귀를 검증.
- 7. CONSTITUTION.md:64 주의: 오케스트레이터가 계약을 OWN하는 것 ≠ 오케스트레이터가 코드를 hand-write. workspace-host.sh(~481줄, osascript injection-safe, IME-safe keycode, debounce ledger)는 substantive 구현 → Rule 4/13에 따라 orchestrator-project coder 세션에 위임 작성. devil's-advocate의 '오케스트레이터=설계만'은 정당하나 해법은 POLICY/PRIMITIVE 분리(primitive를 공유 lib로)이지 OWNERSHIP을 telepty/aterm로 이동이 아님.
